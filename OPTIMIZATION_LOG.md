# MusAgent 优化记录

本文档记录基于最初版本完成的项目整理、功能修复、产品优化和测试工作。以下为压缩版记录，保留所有已做事项，去掉冗长过程说明。

## 优化总览

### 依赖、环境与启动

- **前端依赖补全**
  - 在 `musagent` 中补装并声明前端依赖，重点补齐 `react-router-dom`。
  - 明确当前前端目录为 `musagent`，不是旧目录 `gsap_cocktails`。

- **后端依赖安装**
  - 安装 FastAPI 后端所需依赖，包括 `fastapi`、`uvicorn`、`jieba`、`numpy`、`openai`、`pydantic` 等。
  - 验证后端 `health` 接口可正常返回。

- **本地启动验证**
  - 验证前端可运行在 `http://localhost:5173`。
  - 验证后端可运行在 `http://localhost:8000`。

- **DeepSeek 环境变量说明**
  - DeepSeek API Key 通过环境变量配置：
    ```powershell
    $env:DEEPSEEK_API_KEY="你的 DeepSeek API Key"
    ```
  - 可选配置：
    ```powershell
    $env:DEEPSEEK_BASE_URL="https://api.deepseek.com"
    ```
  - 未配置或调用失败时，后端会降级为算法模板。

### 安全、资源与基础质量

- **npm 安全告警修复**
  - 执行前端依赖安全修复。
  - `npm audit` 最终无漏洞告警。

- **缺失静态资源修复**
  - 补回前端运行所需的 `noise.png`、`mask-img.png` 等静态资源。
  - 避免页面背景、遮罩等视觉资源 404。

- **非阻断提示记录**
  - 记录并区分可忽略提示与真正阻断问题，例如旧目录、构建警告、运行时服务是否启动等。

- **Lint 修复**
  - 清理前端 lint 问题。
  - 后续关键改动均执行 `npm run lint` 验证。

### LLM 调用与生成流程

- **`useLLM` 开关修复**
  - 后端 `run_full_pipeline` 尊重 `useLLM`。
  - `useLLM=false` 时明确跳过 DeepSeek，返回算法模板降级结果。

- **默认启用 DeepSeek WriterAgent**
  - 前端 pipeline 默认传入 `useLLM: true`。
  - 项目默认按 Agent + DeepSeek 的创作流程工作。

- **重新生成接口**
  - 新增 `/api/regenerate`。
  - 支持沿用已有分析结果，只重新执行 WriterAgent，减少重复检索和分析。

### 检索、关键词与情感分析

- **知识库检索升级为 BM25**
  - 将原先简单词集合/相似度检索升级为 BM25。
  - 增加知识库索引缓存，避免每次重复构建。
  - 检索结果返回 `matchedTerms`，方便解释命中了哪些词。

- **关键词权重优化**
  - TF-IDF 引入知识库文档频率。
  - 原始用户输入词增加适度权重。
  - 修复关键词权重平均化、解释性弱的问题。

- **关键词展示优化**
  - 灵感生成页最初将关键词数值规范化为 `0-100` 重要度。
  - 后续进一步改为产品化展示：`高 / 中 / 低`，避免用户误解裸分数。

- **情感强度优化**
  - 修复情感强度长期为 `0` 的问题。
  - 增加主题词对情绪的轻量提示权重。
  - 前端展示从裸数值改为“情绪线索明显 / 中等 / 较弱 / 不明显”。

- **情感分析知识库兜底**
  - 当用户输入缺少直接情绪词时，结合相似作品情绪进行兜底推断。
  - 不额外调用 LLM。

- **低质量关键词过滤**
  - 新增后端 `_is_low_quality_word`。
  - 过滤单字词、停用词、数量词残片、量词残片。
  - 修复 `一张张脸` 被错误切出 `张脸` 并作为关键词的问题。

### 灵感生成模块

- **QueryExpansionAgent**
  - 新增轻量主题扩展逻辑。
  - 对 `校园爱情`、`甜美`、`幸福`、`城市`、`雨夜` 等短主题做拆解和联想扩展。
  - 解决短输入检索不到内容、情绪默认平静的问题。

- **主题理解展示**
  - 灵感生成页增加“主题理解”模块。
  - 展示核心关键词和联想意象，帮助用户理解系统如何解析主题。

- **生成偏好控制**
  - 增加篇幅、语言风格、押韵程度、抽象程度等选项。
  - 后端 prompt 和算法模板均接收这些偏好。

- **结果操作增强**
  - 增加复制结果、沿用分析再生成、重新分析生成、发送到润色。
  - 支持把生成结果写入 `sessionStorage` 并跳转到创作润色页。

- **结果层级优化**
  - 最终生成结果优先展示。
  - Agent 技术细节默认折叠到“Agent 分析详情”。
  - RAG、相似作品、关键词、情绪等信息降低视觉优先级。

- **相似度解释优化**
  - 前端标注 BM25 分数用于排序，不是百分比。

### 创作润色模块

- **润色流程重构**
  - 新增 `/api/polish`。
  - 创作润色不再复用灵感生成 pipeline。
  - 返回原文诊断、润色建议、保守润色版、风格化润色版。

- **润色页面重写**
  - 页面改为专用润色流程。
  - 增加润色目标：文学化、克制、诗性、叙事、更有画面感。
  - 结果分区展示：原文诊断、润色建议、保守润色、风格化润色。

- **LLM 与降级**
  - DeepSeek 可用时执行 LLM 润色。
  - 不可用时使用规则化诊断和轻量改写，不让页面空结果。

### 知识库模块

- **前端标签质量临时修复**
  - 最初在 `MenuPage.jsx` 增加前端展示层过滤。
  - 过滤单字标签，增加文学常用短语兜底。
  - 这是过渡方案，后续已被后端统一接口替代。

- **后端统一知识库接口**
  - 新增 `/api/knowledge`。
  - 后端统一使用 Jieba、TF-IDF、情感逻辑生成知识库卡片数据。
  - 支持分页、搜索、情感筛选、类型筛选。
  - 前端知识库页不再全量加载 5320 首诗并在浏览器中预处理。

- **知识库标签优化**
  - 标签不再大量出现无意义单字。
  - 抽样标签包括 `神经细胞`、`模型飞机`、`博士论文`、`史蒂芬`、`宇宙学` 等短语或实体。

- **知识库页面体验**
  - 保留搜索、筛选、展开全文、加载更多。
  - 增加后端接口不可用提示。

## 当前注意事项

- 后端接口变更后需要重启 FastAPI 服务。
- DeepSeek API Key 未配置或调用失败时，系统会自动降级为算法模板或规则化润色。
- 当前知识库和情感分析仍是轻量统计/NLP 方案，不额外调用 LLM；优点是成本低、响应稳定，缺点是语义抽象能力有限。
